Semantic Badge System

Status badges (green=active, gray=inactive, purple=admin) and audit action pills (green=create, red=delete, blue=update) carry meaning without reading the text. Consistent across all pages.

// PRIORITY ISSUES
P0

No Loading States During Callbacks

When an admin clicks "Save Changes" on the permission matrix, filters users, creates a backup, or takes any callback-driven action, the UI freezes until the response returns. There is no spinner, skeleton, disabled state, or any visual feedback that the system is processing.

FIX: Add loading indicators to all buttons during callback processing. Use Dash dcc.Loading components or button disabled+spinner states. Apply to: filter inputs, save/reset buttons, backup/restore/delete actions, user action buttons.
P1

Action Overload in User Table Rows

Each user row has up to 6 icon buttons (View, Deactivate/Activate, Promote, Reset Password, Delete) all at the same size and visual weight. The delete button (most destructive) is visually identical to the view button until hover. This creates cognitive overhead and increases accidental destructive action risk.

FIX: Keep View as the primary icon button. Move Deactivate, Promote, Reset Password, and Delete into a dropdown menu or expand-on-hover action cluster. Add red color differentiation to destructive actions immediately, not just on hover.
P1

No Confirmation for Destructive Operations

Deactivate user, delete user, delete backup, and restore backup all fire immediately with no confirmation dialog. Only the permission matrix has a confirmation modal. The restore operation is especially dangerous because it triggers an immediate logout and database overwrite.

FIX: Reuse the existing create_modal pattern from admin_roles. Add confirmation modals for: delete user ("Permanently delete [username]?"), deactivate user ("Deactivate [username]'s account?"), delete backup, and restore backup ("This will overwrite the current database and log you out.").
P2

Visual Monotony Across Admin Pages

All four pages use the identical composition pattern: KPI row, section header with green icon, filter toolbar, dark data table. There is no page-specific identity. An admin navigating between Users, Roles, Audit, and Backups has no visual anchor to orient themselves.

FIX: Give each admin page a distinctive layout variation or page-specific accent color. For example: Users could use a card-grid for pending accounts; Audit could use a timeline-style layout; Backups could keep the current two-column but with a visual storage meter.
P2

Audit Filter Toolbar Crowding

Five filter controls (search, action type, user, date range, count label) are crammed into one row. The date range picker is particularly wide and crowds other controls, especially on screens under 1400px.

FIX: Restack the toolbar into two rows: search + action type on top, user filter + date range + count on bottom. Or use a collapsible "Advanced Filters" panel for the date range and user filter.
